# Function List / 功能列表

## 1. Publication workflow / 发表级工作流

- Enforces outline approval before drafting a new full manuscript or review.
- Tracks corpus, evidence map, draft, integrity checks, review, revision, re-review, and final delivery.
- Uses `PASS / WARN / BLOCK` gates to stop unsupported or inconsistent material.

- 新建完整论文或综述时，必须先生成并确认详细大纲。
- 跟踪文献库、证据映射、正文、完整性检查、审稿、修改、再审和最终交付。
- 使用 `PASS / WARN / BLOCK` 阻断缺乏证据或内部不一致的内容。

## 2. Plant-specific writing / 植物领域写作

- Supports original research, narrative reviews, perspectives, scoping reviews, systematic reviews, revisions, rebuttals, and citation audits.
- Preserves species, genotype, tissue, cell type, developmental stage, treatment, stress, environment, and trait boundaries.
- Distinguishes association, prioritisation, necessity, sufficiency, molecular interaction, physiological function, field validation, and breeding value.

- 支持原创研究论文、叙述性综述、观点文章、范围综述、系统综述、稿件修改、审稿回复和引用审计。
- 保留物种、基因型、组织、细胞类型、发育阶段、处理、胁迫、环境和性状边界。
- 区分关联、候选基因优选、必要性、充分性、分子互作、生理功能、田间验证和育种价值。

## 3. Stress-review storyline / 逆境综述叙事框架

- Uses `Why -> What -> Where -> How -> Network -> Integration -> Application -> Future`.
- Integrates abiotic and biotic stress without forcing unsupported categories.
- Maps upstream signals, focal-object regulation, downstream targets, feedback, and adaptive outputs.
- Synthesises hormone networks, stress–stress crosstalk, abiotic–biotic interactions, and development–stress trade-offs.
- Produces an evidence-bounded unified regulatory model.

- 使用 `Why -> What -> Where -> How -> Network -> Integration -> Application -> Future` 主线。
- 在证据允许的范围内整合非生物和生物胁迫，避免机械补齐缺乏证据的类别。
- 梳理上游信号、研究对象调控、下游靶标、反馈回路和适应性输出。
- 综合激素网络、逆境间互作、生物—非生物胁迫互作及发育—逆境权衡。
- 形成具有证据边界的统一调控模型。

## 4. Literature and evidence / 文献与证据

- Records databases, search strings, dates, inclusion/exclusion criteria, deduplication, screening, and evidence appraisal.
- Creates full-paper bilingual close-reading notes with source anchors.
- Maps material claims to sources and grades support.
- Prevents metadata-only records from being used as substantive evidence.
- Audits exact source passages, translations, locations, boundaries, corrections, and retractions.

- 记录数据库、检索式、日期、纳入排除标准、去重、筛选和证据评价。
- 生成带来源定位的全文中英双语精读笔记。
- 将重要论点映射到具体文献并评定支持强度。
- 禁止将只有题录信息的记录作为实质性证据。
- 审核原文证据、翻译、位置、适用边界、勘误和撤稿情况。

## 5. Review synthesis / 综述综合

- Groups evidence by mechanism, scale, method, evidence strength, disagreement, and translation boundary rather than author chronology.
- Reconciles contradictory results through genotype, tissue, stage, stress dose, duration, assay, and environment.
- Requires section-level synthesis claims and explicit transitions.
- Converts future directions into testable programmes and decisive experiments.

- 按机制、尺度、方法、证据强度、争议和转化边界组织研究，而不是按作者年代罗列。
- 从基因型、组织、阶段、胁迫剂量、持续时间、实验方法和环境解释矛盾结果。
- 要求每个章节包含综合论点和明确的逻辑过渡。
- 将展望转化为可检验的研究计划和关键实验。

## 6. Peer review and revision / 审稿与修改

- Uses editor, plant-domain, methods/statistics, genetics/omics/breeding, and devil's-advocate perspectives.
- Tracks every comment through response, manuscript change, location, verification, and final status.
- Supports evidence-based disagreement with incorrect reviewer requests.

- 使用期刊编辑、植物领域、方法统计、遗传组学育种和反方审稿视角。
- 跟踪每条意见对应的回复、正文修改、位置、验证和最终状态。
- 对不合理审稿要求进行有证据支撑的专业回应。

## 7. Figures and application / 图表与应用

- Designs scope, evidence, mechanism/network, unified-model, and future-direction visuals.
- Distinguishes activation, repression, physical interaction, indirect association, and hypothesis.
- Evaluates natural variation, CRISPR, promoter editing, gene pyramiding, omics-assisted breeding, pleiotropy, growth penalties, genotype dependence, and field stability.

- 设计研究范围图、证据表、机制网络图、统一模型图和未来方向框。
- 区分激活、抑制、物理互作、间接关联和假设关系。
- 评价自然变异、CRISPR、启动子编辑、基因聚合、多组学育种、多效性、生长代价、基因型依赖和田间稳定性。

## 8. Deliverables / 交付文件

Completed full projects produce:

1. `01_plant_manuscript_bilingual.docx`
2. `02_peer_review_and_response.docx`
3. `03_literature_audit_and_close_reading.docx`

完整项目最终生成：

1. 中英双语完整稿件。
2. 审稿意见、逐条回复、修改位置和再审结果。
3. 检索记录、引用审计、全文精读笔记及每条引用的原文证据。

The workflow reopens each Word file, checks Heading 1/2/3 hierarchy, renders pages when a renderer is available, visually inspects them, and does not claim visual QA when rendering is unavailable.
